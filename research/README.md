# Research Folder

Research is organized by region so each automation run can build on the existing notes instead of creating duplicate date folders.

- `research/north-america/notes.md`
- `research/europe/notes.md`
- `research/china/notes.md`
- `research/japan/notes.md`
- `research/summary.md`
- `research/benchmark/companies.md`

Each run should:

1. Write all Markdown content, table descriptions, summaries, and final reports in Japanese. Proper nouns, aircraft names, regulation names, source titles, and URLs may keep their original language.
2. Inspect the folder structure and read the current regional notes.
3. Perform fresh web search for North America, Europe, China, and Japan.
4. Compare new sources against the existing notes.
5. Append only new facts, changed regulatory positions, certification milestones, technical progress, or corrected/retired assumptions as rows in the existing `最新動向`, `規制動向`, and `技術動向` tables.
6. Put the research date in the `調査日` column instead of creating a run-specific heading.
7. Update `research/benchmark/companies.md` when a company or aircraft benchmark changes materially.
8. Skip restating unchanged baseline material.
9. Put every source link in `/Volumes/ex/study/e-plane/MOC.md`.
